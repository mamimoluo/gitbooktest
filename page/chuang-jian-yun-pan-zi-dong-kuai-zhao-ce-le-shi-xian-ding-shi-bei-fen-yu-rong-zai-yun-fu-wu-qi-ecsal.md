文章内容

通过自动快照策略，系统将周期性地为指定的云盘创建快照，避免因人为疏忽忘记创建快照，降低数据丢失风险，提高系统的可靠性和稳定性。同时，支持设置跨地域复制快照，即使发生地域性的灾难或故障，仍可在其他地域快速恢复数据。本文介绍如何创建自动快照策略。

### 前提条件

已开通快照服务。具体操作，请参见 [开通快照](/broken/pages/e3c74b879c31e0f706fca27967fc90accb681b26)。

### 操作步骤

说明：如果您在当前地域没有任何快照策略，系统将在您首次为云盘指定策略时创建默认策略（default\_policy），您可以根据业务需求使用默认策略或者创建自定义策略。

{% stepper %}
{% step %}
### 步骤

1. 访问 ECS 控制台 - 自动快照策略： https://ecs.console.aliyun.com/autoSnapshotPolicy/region
{% endstep %}

{% step %}
### 步骤

2. 在页面左侧顶部，选择目标资源所在的资源组和地域。

![地域](../.gitbook/assets/p680076.png)
{% endstep %}

{% step %}
### 步骤

3. 在“自动快照策略”页签中单击“创建自动快照策略”。
{% endstep %}

{% step %}
### 步骤

4. 在“创建自动快照策略”页面，设置自动快照策略参数，并单击“确定”。
{% endstep %}

{% step %}
### 步骤

5. 自动快照策略创建完成后，建议立即为云盘设置该策略，实现云盘自动创建快照备份数据的能力。

具体操作，请参见 [为云盘设置自动快照策略](/broken/pages/bccc79572786aef69d81f25851c820dbfad07326#concept-nyv-k3l-xdb)。
{% endstep %}
{% endstepper %}

#### 策略信息

设置自动快照策略的名称、执行时间、快照保留时间等信息。

| 参数   | 说明                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ---- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 策略名称 | 自定义自动快照策略的名称。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| 重复日期 | 创建自动快照的日期，支持在周一至周日之间选择一个或多个日期。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| 开始时间 | <p>- 自动快照的创建时间点。使用UTC+8时间，单位为小时。<br><br>- 自动快照将在开始时间后的一个小时内进行创建。<br><br>- 当一天内需要创建多次自动快照时，可以在00:00~23:00共24个整点中选择多个时间点。<br><br>重要：创建快照会暂时降低块存储I/O性能，一般性能差异在10%以内，可能会出现短暂瞬间变慢。建议您选择避开业务高峰的时间点。</p>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| 保留时间 | <p>设置自动快照的保留时间。系统默认持续保留，您也可以自定义时长。<br>- 自定义时长：快照超过自定义的保留时间后，将被系统自动删除。<br><br>- 持续保留，直至快照数量达到额度上限后被自动删除：在自动快照数量达到上限后，系统会自动删除最早创建的自动快照。<br><br>说明：<br>- 选择自定义保留时间后，当快照的保留时间还剩3天时，快照列表的“保留时间”列会高亮提示“xx天后自动释放”。您可以根据需要在快照到期前延长快照的保留时间，操作请参见 <a href="/broken/pages/33e4b8fbab51f8173b76324ad0a56e19faa3f008">延长快照保留时间</a>。<br><br>- 自动快照的额度上限，请参见 <a href="user-guide/yun-pan-shu-ju-shi-jian-dian-bei-fen-hui-fu-kuai-zhao-yun-fu-wu-qi-ecsali-yun.md#section-bky-2dp-c5p">使用限制</a>。<br><br>- 快照策略创建完成后，后续您可以通过 <a href="/broken/pages/5c160be49797695c1214507f06a8c1a079273cbb">修改自动快照策略</a> 调整保留时间。调整保留时间不影响已有的快照，只会对新创建的自动快照生效。例如已有快照保留时间是20天，修改快照策略后快照保留时间是10天，新自动快照策略不会影响已有快照的保留时间。<br><br>- 如果 <a href="/broken/pages/bccc79572786aef69d81f25851c820dbfad07326#1abb5dfb98d4a">设置自动快照随云盘释放</a> 属性，当释放云盘（手动释放云盘、云盘随实例释放或更换系统盘）时，即使快照未到期，自动快照也会随云盘释放而提前删除。<br><br>- 无论快照是自定义保留时间还是永久保留，您可以按需删除快照，避免持续产生快照存储费用。具体操作，请参见 <a href="/broken/pages/e5d973f2b90412794ad416b182fa9fe482b48fa9">删除快照</a>。</p> |

#### 跨地域复制策略

重要提示：

* 快照复制后，会在目标地域生成一份快照副本，因此会产生跨地域的流量费用。同时，目标地域的快照副本会根据容量与存储时间收取快照存储费用。更多信息，请参见 [快照计费](/broken/pages/817a93eb5a881a1fc26c8385183752f8c7a3e0a7#concept-rq2-pcx-ydb)。
* 关于复制快照更多信息，请参见 [复制快照](/broken/pages/1c500731fff30ca48c1a5b0c8efcae026e9d122f)。

| 参数        | 说明                                                                                                                                                                                                                                                                      |
| --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 快照跨地域复制   | 启用后，通过该自动快照策略创建的快照将自动复制到目标地域，以实现数据的备份和容灾。                                                                                                                                                                                                                               |
| 目标地域      | 选择快照需要复制到的目标地域。                                                                                                                                                                                                                                                         |
| 复制快照的保留时间 | 复制的副本快照在目标地域中的保留时间。                                                                                                                                                                                                                                                     |
| 加密复制      | <p>- 不启用：如果您的源端快照为加密快照，系统将会使用密钥管理服务KMS在目标地域创建的服务密钥将快照加密复制到目标地域；如果您的源端为非加密快照，则直接复制到目标地域。<br><br>- 启用：无论您的源端快照是否为加密快照，均可以将创建的自动快照基于指定的密钥加密复制到目标地域。<br><br>关于ECS加密密钥的更多说明，请参见 <a href="/broken/pages/3bae502f9f663f1b3dffcd6a2a9fa1af8e2c1cd3#b0d1bf746d8cf">加密密钥</a>。</p> |
